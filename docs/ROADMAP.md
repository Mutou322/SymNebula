# SymNebula 开发规划

## 第一阶段：逻辑奇点 (The Logical Singularity)
目标：实现单个中空神经元的"公式硬解码"与"双向约束求解"。

### 节点 1.1：公式解析引擎 (Symbolic Parser)
- 使用 pest 或 nom 库编写数学语法
- 将字符串（如 x^2 + y = 10）解析为 AST
- 识别变量占位符，自动提取变量名作为"突触接线柱"
- **当前状态：已完成（极简版手写解析器，后续恢复 nom）**

### 节点 1.2：约束平衡器 (Local Solver)
- 实现初级代数变形逻辑
- 输入变量增加时，尝试对等式移项求解
- 无法代数求解时引入数值迭代算子（如牛顿法）
- **当前状态：已完成（加减乘除幂的代数重排）**

### 节点 1.3：状态机原型 (State Machine)
- 定义四种状态（灰/黄/绿/紫）的切换逻辑
- 建立维度检查逻辑，矩阵乘法不匹配时立即触发紫色
- **当前状态：已完成（Gray/Yellow/Green/Purple）**

## 第二阶段：星云拓扑 (The Nebula Topology)
目标：实现多节点链接、作用域隔离与数据流转。

### 节点 2.1：去中心化图结构 (Graph Core)
- 利用 petgraph 或自定义 SlotMap 管理节点和边
- 实现"突触"连接，确保 NodeA 输出映射到 NodeB 输入内存地址
- **当前状态：已完成（纯 std Vec 实现）**

### 节点 2.2：符号隔离与镜像 (Scoping)
- 每个节点独立 GUID 与局部变量名映射
- 实现"参数镜像"节点，全局常量同步
- **当前状态：待开发**

### 节点 2.3：依赖分析 (Dependency Analysis)
- 内核自动扫描无依赖节点和有环路节点
- **当前状态：已基础实现（step() 中通过 env 传播）**

## 第三阶段：生命律动 (The Pulse & Dynamics)
目标：引入 Tick 机制，解决反馈环路与并行调度。

### 节点 3.1：逻辑时钟调度器 (Tick Scheduler)
- 实现 T_n = f(T_{n-1}) 的迭代逻辑
- 每 Tick 锁定上一时刻数据，并行计算当前时刻活跃节点
- **当前状态：已完成（基础 step() 调度器）**

### 节点 3.2：Rust-C 算子桥接 (Computation Layer)
- 将矩阵分解、ODE 求解等封装进底层 C 库，由 Rust 调用
- **当前状态：未开始**

### 节点 3.3：并行加速 (Parallelism)
- 利用 Rayon 库，根据依赖图将计算分配到 CPU 所有核心
- **当前状态：未开始（第一版曾用 Rayon，纯 std 版已移除）**

## 第四阶段：感知边界 (The Sensory Boundary)
目标：建立 Python UI 层与 Rust 内核的"最薄"接口。

### 节点 4.1：高性能数据通道 (Data Bridge)
- 使用 PyO3 或共享内存，Python 低延迟读取 Rust 数据
- **当前状态：未开始（ui/ 骨架文件占位）**

### 节点 4.2：星云编辑器 (Nebula Editor)
- 使用 PyQt/PySide 实现图形化拖拽
- 集成 LaTeX 编辑组件
- **当前状态：未开始**

### 节点 4.3：观察窗与投影协议 (Observation Window)
- 使用 VisPy/PyOpenGL 渲染 3D 空间
- Python 端定义投影公式，Rust 映射高维数据为坐标
- **当前状态：未开始**

## 第五阶段：封装与分发 (The Ecosystem)
目标：实现 .brainstorm 格式与 U 盘免安装运行。

### 节点 5.1：序列化与 .brainstorm
- AST 和图结构的 XML 序列化，打包为 ZIP
- **当前状态：未开始**

### 节点 5.2：静态预扫描安全机制
- 打开项目时预判计算深度，防止"计算炸弹"
- **当前状态：未开始**

### 节点 5.3：便携化打包
- PyInstaller + Rust 静态编译，100MB 内单文件
- **当前状态：未开始**
